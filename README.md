# Artbuild

**Artbuild** — концепция спокойной мобильной arcade/creative-building игры про создание живой вертикальной башни-города из маленьких красивых домиков.

Игрок не строит сложные конструкции вручную и не занимается тяжёлым менеджментом. Он постепенно выбирает из нескольких предложенных блоков, размещает их на башне, назначает им функцию, обеспечивает энергией и наблюдает, как башня превращается в уникальный архитектурный арт-объект.

## Главная идея

Игрок строит не просто башню, а **живой вертикальный город**. Каждый маленький домик — часть общей композиции: жилой блок, кафе, библиотека, мастерская, лаборатория, художественная студия, магазин, ветряк, балкон, мостик или другой декоративно-функциональный элемент.

Основной акцент игры:

- красивый визуальный рост башни;
- спокойный темп без давления;
- простые решения вместо сложного строительства;
- баланс жителей, энергии, красоты и технологии;
- уникальная башня у каждого игрока;
- возможность делиться ростом башни в формате коротких видео.

## Базовый игровой цикл

```text
Выбор домика → Размещение → Назначение функции → Заселение жителей → Энергия и баланс → Новые домики
```

Игрок получает на выбор, например, три новых домика или блока. Он выбирает понравившийся, размещает его сверху, по центру, слева, справа или в доступном месте башни. После этого домик может получить назначение: жилой, кафе, библиотека, мастерская, лаборатория, арт-студия и так далее.

## Первый runtime-прототип

Первый технический каркас сделан как **engine-agnostic JavaScript prototype**. Это не финальный выбор движка, а способ зафиксировать модульные границы до привязки к Unity, Godot или другому мобильному runtime.

Локальные команды:

```bash
npm run smoke
npm run validate:content
npm run test:placement
npm run test:playable-loop
npm run test:choice
npm run test:energy
npm run test:residents
npm run test:progression
npm run prototype:loop
npm test
npm start
```

`npm run smoke` проверяет, что приложение импортируется и что базовые слои проекта зарегистрированы. `npm run validate:content` проверяет первый каталог домиков. `npm run test:placement` проверяет доменную логику слотов и размещения. `npm run test:choice` проверяет выбор из трёх вариантов. `npm run test:energy` проверяет powered/unpowered состояние. `npm run test:residents` проверяет жителей и комфорт. `npm run test:progression` проверяет красоту, технологию и открытия зданий. `npm run prototype:loop` запускает первый текстовый construction loop.

## Принцип разработки

Проект должен строиться модульно. Игровая логика, визуальное представление, данные ассетов, прогрессия, энергия и UI не должны смешиваться в один монолитный класс или сцену. Каждая система должна иметь свою область ответственности и возможность расширяться независимо.

Главная техническая цель первого этапа: научиться строить основание и постепенно «лепить» на него разные домики так, чтобы они стыковались друг с другом и создавали красивую башню в духе визуальных референсов.
